故事開場

場景：Yui 的揭露兩難

SCENE · 2026 · Kyoto Univ.

三種 AI、三種揭露規則

博士後 Yui 正在準備她的第一篇 corresponding author 論文，研究單細胞圖譜。她用了三個 AI 工具：ChatGPT-5 幫她在 Discussion 段落修語法、潤飾英文表達；Claude Sonnet 4.6 幫她重構分析 pipeline 的 Python code (寫 wrapper、加 docstring)；以及 DALL-E 生成一張漂亮的 graphical abstract——藍色細胞群、發光的標記基因，看起來像 Cell 封面。

投稿前一週她問 PI Sato 教授：「我要不要全部揭露？」Sato 教授翻出三個期刊政策頁：「三種工具、三種規則。(1) ChatGPT 純粹潤飾語法——Elsevier 政策說這類使用「鼓勵但非強制揭露」，你可以選擇寫一句聲明；(2) Claude 寫了實質的 code——這必須在 Methods 揭露，並在 Code Availability 註明 AI 協助；(3) DALL-E 生成的 graphical abstract——Cell Press 明文禁止 generative AI 生成科學圖像，conceptual abstract 是灰色地帶但必須揭露。建議你直接換成手繪或 BioRender。」

Yui 重寫 disclosure section、把 graphical abstract 換成 BioRender，然後同步上傳到 bioRxiv——24 小時內拿到 DOI。三個月後論文被接受，reviewer 還特別讚許「disclosure transparency exemplary」。Yui 學到了：AI 是工具，揭露是責任，每個工具的政策都不一樣，必須一一查清楚。

Postdoc Yui is preparing her first corresponding-author paper, a single-cell atlas study. She used three AI tools: ChatGPT-5 to polish English grammar in her Discussion, Claude Sonnet 4.6 to refactor her Python analysis pipeline (wrappers, docstrings), and DALL-E to generate a stunning graphical abstract — blue cell clusters, glowing marker genes, Cell-cover quality.

A week before submission she asks her PI Prof. Sato: "Do I disclose all of it?" Sato pulls up three policy pages: "Three tools, three rules. (1) ChatGPT for pure grammar polish — Elsevier's policy calls disclosure 'encouraged but optional' for this use; you can include a statement if you want. (2) Claude wrote substantive code — this MUST be disclosed in Methods and noted in Code Availability. (3) DALL-E graphical abstract — Cell Press explicitly bans generative AI for scientific images; conceptual abstracts are grey but require disclosure. Better: redraw it in BioRender."

Yui rewrites the disclosure section, replaces the graphical abstract with BioRender, and submits to bioRxiv simultaneously with the journal — DOI assigned within 24 hours. Three months later the paper is accepted; one reviewer commends the "exemplary disclosure transparency." Yui learned: AI is a tool, disclosure is responsibility, every tool has different rules — check every one.

逐步推導

現代可重現投稿 8 步

  1. Σ₁ — Day 1 建立 Git repo + uv (Python) / renv (R) lockfile：環境鎖死，後人可重建。Day 1 set up Git repo + uv (Python) / renv (R) lockfile: lock the environment so others can rebuild it.
  2. Σ₂ — 用 Snakemake / Nextflow 建立 pipeline：每一步輸入 / 輸出明確、可重跑、可平行。Build the pipeline in Snakemake / Nextflow: explicit inputs/outputs per step, rerunnable, parallelizable.
  3. Σ₃ — 寫作過程同步記錄 AI 使用：日記式記下「哪段用了哪個工具、哪個版本、做了什麼」。Document AI usage as you go: keep a journal — "which section, which tool, which version, doing what."
  4. Σ₄ — 用 Quarto (.qmd) 寫稿：文字 + code + 圖表整合，所有 figures 由 code 直接產生。Write in Quarto (.qmd): text + code + figures integrated; all figures generated directly from code.
  5. Σ₅ — 用 Docker 容器化分析環境：Dockerfile 公開、image push 到 Docker Hub / GitHub Registry。Docker-containerize the analysis env: Dockerfile public, image pushed to Docker Hub / GitHub Registry.
  6. Σ₆ — 投稿日同步上傳 bioRxiv：24 小時內取得 DOI，搶優先權、加速傳播。On submission day, post to bioRxiv simultaneously: DOI within 24 h, secures priority, speeds dissemination.
  7. Σ₇ — 在 Methods 或 Acknowledgements 加 AI disclosure statement：工具 + 版本 + 用途 + 段落。Add an AI disclosure statement in Methods / Acknowledgements: tool + version + purpose + section.
  8. Σ₈ — 論文接受後在 GitHub 打 release tag 並串接 Zenodo 取得永久 DOI——code 從此可被引用且不消失。After acceptance, tag a GitHub release and link to Zenodo for a permanent DOI — code becomes citable and never disappears.
專家辯論

三個尚未有共識的問題

(a) AI 生成的科學會取代人類研究嗎？

支持：AlphaFold 等 AI 已能完成過去博士論文等級的任務；2025 起 "AI scientist" papers 出現。反對：AI 缺少 falsifiability 直覺與物理實驗能力；目前仍是 amplifier，非 replacer。

Pro: Systems like AlphaFold now complete what used to be PhD-level tasks; "AI scientist" papers emerged from 2025. Con: AI lacks falsifiability intuition and wet-lab capability; remains an amplifier, not a replacer.

(b) 期刊該不該主動驗證 AI 揭露？

支持：目前是 honor system，違規無法稽核；驗證可提升 transparency。反對：AI 偵測工具誤判率高 (Liang 2023)；可能誤殺非英語母語作者。

Pro: Current honor system has no audit; verification would boost transparency. Con: AI-detection tools have high false-positive rates (Liang 2023) and may unfairly flag non-native English speakers.

(c) Preprint 會取代 peer review 嗎？

支持：2020 COVID-19 期間 bioRxiv preprint 影響力超越期刊論文；社群審閱模式興起 (eLife 2023)。反對：正式 peer review 仍是科學 gatekeeping 的核心；preprint 並非品質保證。

Pro: During COVID-19 in 2020, bioRxiv preprints had more impact than journal papers; community-review models rose (eLife 2023). Con: Formal peer review remains core to scientific gatekeeping; preprints are not quality-assured.

聚光燈

一個典範、一個警鐘

MODEL：2020 SARS-CoV-2 與 bioRxiv 全球響應

2020 年 COVID-19 爆發初期，從病毒序列、結構、流病到疫苗候選，幾乎所有關鍵論文都先以 bioRxiv / medRxiv preprint 公開——以致全球研究者可以在「peer review 仍進行中」時就基於最新資料展開後續工作。沒有 preprint，COVID-19 的科學響應速度至少會慢 6-12 個月。這是 preprint 文化的最佳示範。

In early 2020, nearly every key COVID-19 paper — viral sequence, structure, epidemiology, vaccine candidates — first appeared on bioRxiv / medRxiv as preprints, letting global researchers build on the latest data while peer review was still in progress. Without preprints, the scientific COVID-19 response would have been at least 6-12 months slower. The best demonstration of preprint culture.

WARNING：2023 ChatGPT 被列為 author 的論文事件

2023 年若干早期 preprint 與少數期刊論文將 ChatGPT 列為 co-author——被讀者與其他研究者迅速指認，期刊收回 / 退稿並要求撤銷 author 身份。事件直接催生 ICMJE 2023 更新與 Nature、Science、Cell 全面禁令——任何主流期刊 2026 都不再容許 AI 為 author。

In 2023 several early preprints and a handful of journal papers listed ChatGPT as a co-author — quickly identified by readers and other researchers; journals retracted / rejected and demanded author removal. The incident directly triggered the ICMJE 2023 update and universal bans by Nature, Science, Cell — by 2026 no mainstream journal allows AI as author.

陷阱

6 個典型陷阱

① 把 ChatGPT / Claude 列為 co-author

2026 所有主流期刊明文禁止；違反 = 直接退稿或撤稿。AI 無法承擔 ICMJE 第 ④ 條 accountability。

All 2026 mainstream publishers explicitly prohibit this; violation = desk reject or retraction. AI cannot bear ICMJE criterion ④ accountability.

② 同步投 bioRxiv 給禁止 preprint 的期刊

2026 已罕見但仍存在（部分 NEJM 子題、少數臨床期刊）。投稿前查 SHERPA/RoMEO 與 journal preprint policy。

Rare by 2026 but still exists (some NEJM subtypes, a few clinical journals). Check SHERPA/RoMEO and the journal's preprint policy before submitting.

③ 該揭露的 AI 使用卻沒揭露

用 AI 寫實質段落 / 寫 code 不揭露 = 違反期刊政策。揭露不會被退稿，不揭露被抓會。

Using AI for substantive content / code without disclosure = policy violation. Disclosure doesn't get you rejected; failure to disclose, if caught, does.

④ 忘了串 Zenodo 給 code DOI

GitHub repo 隨時可能被刪除；論文裡只貼 GitHub link 等於 5 年後 broken link。串 Zenodo = 永久 DOI。

GitHub repos can be deleted any time; a paper that only cites a GitHub link is a broken link in 5 years. Link to Zenodo for a permanent DOI.

⑤「Data available on request」

2026 多數主流期刊已不接受此描述；要求實際 deposit (GEO, ENA, dbGaP, Zenodo) 並提供 accession。

Most major 2026 publishers no longer accept this phrasing; require actual deposit (GEO, ENA, dbGaP, Zenodo) with accession.

⑥ Reviewer 把 manuscript 上傳公開 AI

Confidentiality breach；違反 peer review 倫理。Cell, NEJM, JAMA, NIH 明令禁止。違反者可能被 ban 未來審稿。

Confidentiality breach; violates peer-review ethics. Cell, NEJM, JAMA, NIH explicitly prohibit. Violators may be banned from future reviewing.

❌ 不好的揭露

"We used AI in this study."

沒說工具、沒說版本、沒說用途、沒說段落——等於沒揭露。

No tool, no version, no purpose, no section — equivalent to no disclosure.

✅ 好的揭露

"During the preparation of this manuscript, we used ChatGPT-5 (OpenAI, Jan 2026) to improve language and check grammar in the Discussion section. Authors reviewed and edited the content and take full responsibility."

工具 ✓ 版本 ✓ 用途 ✓ 段落 ✓ 負責聲明 ✓——範本級揭露。

Tool ✓ version ✓ purpose ✓ section ✓ accountability ✓ — template-level disclosure.

自我反思 6 問

  1. 你每一個 AI 使用都揭露了嗎？
  2. 你的 code 是否同時放上 GitHub + Zenodo？
  3. 陌生人能不能在 1 個月內重現你的 figures？
  4. 你的 raw data 已 deposit 到適當 repository 並有 accession？
  5. 你的 disclosure 包含工具 / 版本 / 用途 / 段落 / 負責聲明？

進階

2024-2026 SOTA：政策、工具與工作流

(a) 2026 主流出版商 AI 政策比較

Elsevier：AI 工具不可為 author；用於草擬實質內容必揭露；純文法 / 翻譯例外。Springer Nature：類似 Elsevier，但對 AI 生成 figure 採取「個案審查」。Cell Press：最嚴格，generative AI 不可生成 / 修改科學圖；conceptual graphical abstract 灰色地帶且需揭露。APA：建議所有 AI 使用揭露（含文法），社會科學取向。Taylor & Francis：跟隨 ICMJE；要求 author 對所有 AI 生成內容負完全責任。

Elsevier: AI tools cannot be authors; substantive use must be disclosed; pure grammar / translation exempt. Springer Nature: Similar to Elsevier, but AI-generated figures undergo "case-by-case review." Cell Press: Strictest; generative AI cannot create / alter scientific images; conceptual graphical abstracts are grey and require disclosure. APA: Recommends disclosing all AI use (including grammar), social-science-leaning. Taylor & Francis: Follows ICMJE; authors fully accountable for all AI-generated content.

(b) Quarto 2024-26 的互動式發表

Quarto v1.4+ (2024) 支援 Observable JS、Shinylive、Pyodide 直接 in-browser 執行 Python / R；單一 .qmd 可同時輸出靜態 PDF / HTML、互動 dashboard、書、簡報。2026 已成為計算科學社群的事實標準，逐步取代 R Markdown + bookdown + xaringan 等獨立工具。

Quarto v1.4+ (2024) supports Observable JS, Shinylive, Pyodide for in-browser Python / R execution; a single .qmd can output static PDF / HTML, interactive dashboards, books, and slides. By 2026 it is the de facto standard in computational science, gradually replacing R Markdown + bookdown + xaringan.

(c) Snakemake 8+ vs Nextflow 24+

Snakemake 8 (2025)：Python-based、rules + wildcards、conda / container 整合、學術社群偏好；新版執行器 plugin 化、支援 Kubernetes / SLURM 雲端跑。Nextflow 24 (2024)：Groovy DSL2、nf-core 社群超強大 (2026 已有 100+ 標準 pipeline)、企業 / 臨床傾向、tower.nf 商業支援。選擇取決於團隊背景與 pipeline 性質——學術小群偏 Snakemake，跨機構 / 臨床傾 Nextflow。

Snakemake 8 (2025): Python-based, rules + wildcards, conda / container integration, academic-community preferred; new pluggable executors support Kubernetes / SLURM / cloud. Nextflow 24 (2024): Groovy DSL2, nf-core community very strong (100+ standard pipelines by 2026), enterprise / clinical-leaning, tower.nf commercial support. Choice depends on team and pipeline nature — small academic groups lean Snakemake; cross-institution / clinical lean Nextflow.

歷史小故事：arXiv 從 10 位物理學家開始

1991 年，Paul Ginsparg 在 Los Alamos National Laboratory 為大約 10 位高能物理同行架了一個私人 preprint email server，目的是讓大家不用等期刊就能交換最新研究草稿。他完全沒想到這會改變整個科學。35 年後的 2026，arXiv 累積超過 240 萬篇 preprint，並衍生出 bioRxiv (2013, CSHL)、medRxiv (2019, CSHL+Yale+BMJ)、chemRxiv、SocArXiv 等姊妹站。從 10 位物理學家到全球科學的標準工作流——一個人的小工具，改寫了整個科學發表生態。

In 1991, Paul Ginsparg set up a private preprint email server at Los Alamos National Laboratory for about 10 high-energy physics colleagues so they could exchange research drafts without waiting for journals. He had no idea it would change all of science. 35 years later, in 2026, arXiv has accumulated over 2.4M preprints and spawned bioRxiv (2013, CSHL), medRxiv (2019, CSHL+Yale+BMJ), chemRxiv, SocArXiv, and more. From 10 physicists to a global scientific standard — one person's small tool rewrote the entire publishing ecosystem.

From 2023 to 2026, the academic writing toolchain saw three shifts: (1) generative AI (ChatGPT, Claude, Gemini) entered the workflow; (2) preprints went from optional to standard (bioRxiv exceeds 300k papers); (3) Reproducibility tools (Quarto, Snakemake, Docker) shifted from "nice to have" to "journal-required."

This page covers what 2026 mainstream journals actually require, what they prohibit, and how to use AI compliantly.

2026 AI 政策

一、AI 政策共識

截至 2026/05，Elsevier, Springer Nature, Wiley, Taylor & Francis, SAGE, Cell Press 等主流期刊的 AI 政策已收斂為以下五點。

As of 2026/05, AI policies across Elsevier, Springer Nature, Wiley, Taylor & Francis, SAGE, Cell Press, and other major publishers have converged to the following five points.

政策說明
① AI 不能掛 author 所有主流期刊禁止 ChatGPT, Claude, Gemini 等列為 author 或 co-author。理由：AI 無法承擔 ICMJE 第 ④ 條 "accountability"。All major publishers prohibit ChatGPT, Claude, Gemini etc. as authors. Rationale: AI cannot bear ICMJE criterion ④ "accountability".
② Generative 內容必須揭露 若 AI 用於草擬實質內容，必須在 Methods 或 Acknowledgements 揭露：工具名稱 + 版本 + 用途 + 用在哪個段落。If AI drafted substantive content, disclose in Methods or Acknowledgements: tool name + version + purpose + where used.
③ 文法/翻譯/校對例外 用 Grammarly, DeepL, Claude 純粹做語法 / 翻譯 / 校對，多數期刊不要求揭露 (但 APA、Cell 仍鼓勵揭露)。Using Grammarly, DeepL, Claude purely for grammar/translation/proofreading typically doesn't need disclosure (but APA, Cell still encourage it).
④ 圖片生成/變造嚴禁 Cell Press 等明文禁止 generative AI 製作或修改科學圖片 (DALL-E, Midjourney, Stable Diffusion)。Conceptual graphical abstract 可能例外但要揭露。Cell Press etc. explicitly prohibit generative AI for creating/altering scientific images (DALL-E, Midjourney, Stable Diffusion). Conceptual graphical abstracts may be exempt but require disclosure.
⑤ Reviewer 不得用 AI Cell, NEJM, JAMA, NIH 明文禁止 reviewer 把 confidential manuscript 上傳到 ChatGPT 等公開 AI (洩漏 confidential research)。違反 = peer review breach。Cell, NEJM, JAMA, NIH explicitly prohibit reviewers from uploading confidential manuscripts to public AI (ChatGPT, Claude). Doing so = peer review breach.
⚠️
2026 的灰色地帶：(a) 用 AI 做 literature review 摘要——多數期刊不要求揭露，但要求 author 對引用的所有事實負責（AI 常 hallucinate citations！）；(b) 用 AI 做 Statistical analysis code——若 AI 寫了 code 然後 author 跑、debug、解釋，多數期刊視為 "tool use" 不要求揭露 (但要在 Code Availability 註明)。
2026 grey zones: (a) Using AI to summarize literature — most journals don't require disclosure but you remain accountable for every cited fact (AI hallucinates citations!); (b) Using AI to write statistical code — if AI drafts and author runs/debugs/explains, most journals treat as "tool use" without disclosure (but note in Code Availability).

三、Preprint 工作流

2026 年 preprint 已成主流。Cell Press, Nature, PNAS, eLife 等主刊均明確支援 preprint 與後續正式發表並行 (post-bioRxiv 不被視為 prior publication)。

建議流程：

  1. 論文寫好後 → 投 bioRxiv (生命科學) / medRxiv (臨床) / arXiv (計算/物理) / Research Square (跨領域)。
  2. 同時 / 之後投正式期刊。

By 2026, preprints are mainstream. Cell Press, Nature, PNAS, eLife etc. explicitly support preprints alongside formal publication (post-bioRxiv ≠ prior publication).

Recommended workflow:

  1. Draft complete → post to bioRxiv (life sciences) / medRxiv (clinical) / arXiv (computational/physics) / Research Square (interdisciplinary).
  2. Simultaneously / subsequently submit to a formal journal.
  3. Preprint DOI is immediately citable — useful for grant applications, conference talks, recruitment.
  4. Once accepted, bioRxiv auto-links to the formal published version.
